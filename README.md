# Fact Archive

학습한 기술 지식 중 **객관적인 사실(Fact)**만을 모아두고, AI를 통해 복습하는 저장소입니다.

---

## 목표

**완성도와 복습 효율**을 둘 다 챙기기 위해 다음을 추구합니다:

- **사실(Fact) 우선**: 개인적 의견이 아닌 공식 문서 기반의 객관적 지식만 기록
- **출처 명시 필수**: 모든 내용은 검증 가능한 출처와 함께 저장
- **AI 주도 복습**: 시니어 면접관 페르소나를 통한 엄격한 기술 검증
- **체계적 관리**: 표준화된 양식과 자동화된 워크플로우

---

## 접근 방법

### 구조화된 instruction 시스템

모든 규칙과 워크플로우를 마크다운 파일로 구조화했습니다.
AI가 필요한 시점에 필요한 문서만 로드할 수 있도록 설계했습니다.

**instruction-map.md**가 라우팅 역할을 합니다.
사용자가 "복습하자"라고 하면 AI가 자동으로 playbook/review.md를 찾아 로드합니다.

### AI 주도 설계

**"사용자가 규칙을 완벽히 기억하는 게 아니라, AI가 규칙을 기억하고 사용자를 리드하는 것"**

규칙이 많아지면 사용자가 모든 걸 완벽히 기억하는 건 불가능합니다. 따라서:

- **AI가 주도적으로 안내**: 다음 단계 제안, 적절한 가이드 자동 로드
- **잘못된 요청 제지**: 사용자가 비효율적이거나 잘못된 방향으로 가려 할 때 AI가 제지하고 대안 제시
- **사용자는 편하게**: 모든 디테일을 기억할 필요 없이, AI의 안내를 따라가기만 하면 됨

## 프로젝트 구조

```
fact-archive/
├─ instruction-map.md    # 문서 라우팅 맵
├─ knowledge/            # 지식 보관소 (객관적 사실만)
├─ playbook/             # 활용 가이드 (복습 등)
└─ meta/                 # 프로젝트 관리 가이드
   ├─ agent.md           # AI 전역 규칙
   ├─ instructions/      # 작업별 가이드
   └─ rules/             # 공통 규칙 (재사용)
```

## 사용법

Claude Code나 다른 AI 에이전트에서 작업을 요청하면 됩니다.

```
"React Hooks 공부한 내용 정리해줘"
"네트워크 관련 질문으로 복습하자"
"구글 문서를 fact-archive 형식으로 변환해줘"
```

AI가 instruction-map.md를 참조해서 적절한 문서를 찾아 로드합니다.

자세한 내용은 [instruction-map.md](./instruction-map.md)를 참고해주세요.
